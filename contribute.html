Mitmachen

Wie du beitragen kannst

  • Module verbessern: Formulierungen schärfen, Beispiele ergänzen, Querverknüpfungen prüfen.
  • Bugs melden: Fehler in UI/Links/Offline-Modus dokumentieren.
  • Neue Ressourcen: Quellenlisten, Anleitungen, Übersetzungen.

Alles läuft offen – bitte beachte die Lizenz (CC BY-SA + Dignity Clause).

Projektstruktur (kurz)

/
├─ index.html · Start
├─ map.html + map.js · Karte
├─ modules.html + modules.js · Liste/Suche
├─ module.html + module.js · Detailseite
├─ MODULES/modules.json · Datenquelle
└─ sw.js, nav.js, styles.css …

Beitragswege

  1. Issue eröffnen: Beschreibe Problem/Verbesserung präzise, ggf. Screenshot.
  2. Pull Request: Forken, Branch erstellen, Änderungen mit knappen Commits, PR eröffnen.
  3. Diskussion: Größere Ideen erst als Issue/Discussion skizzieren.

Style: klar, inklusiv, faktenbasiert; keine personenbezogenen Daten in Beispielen.

Guidelines

  • Konsistenz: Schreibweisen exakt wie STATIC CORE V5.0 LOCKED (z. B. PSYCHE, TRUTH, MEDIA).
  • Kurzbeschreibungen: 2–3 Sätze + optional Negativszenario in einem Satz.
  • Links-Matrix: Pro Modul 2–3 Kern-Verknüpfungen, begründet.
  • Barrierefreiheit: A11y beachten (Kontraste, aria-labels, Tastatur-Nutzung).
